Heartland Storm Guide Site Plan

An informational website for Tornado Alley safety, learning, and simple tools.

Site Name

Heartland Storm Guide – clear, memorable, and relevant to Tornado Alley’s geography and purpose (storm preparedness & education). It communicates both the region (Heartland) and the function (a guide).

Optional domain availability: heartlandstormguide.com / .org / .net (not required for this course; final choice TBD).

Site Purpose

Provide a friendly, accessible hub for severe-weather literacy and safety in Tornado Alley. The site will:

  • Explain severe storm basics with concise, visual guides (supercell types, radar signatures, watch vs warning, go-bag).
  • Offer simple web tools (wind-chill & heat-index calculators; saved “favorite locations” via localStorage).
  • Enable a client-validated Severe Weather Report form (name, location, hazard type, notes) with a confirmation message.
  • Model good web practices: responsive layout, strong contrast, semantic HTML, and performance optimizations (lazy loading, optimized images).

Scenarios (Target Audience Questions)

  • “How do I tell an HP from an LP supercell, and why does it matter for safety?”
  • “What’s the difference between a Tornado Watch and a Tornado Warning?”
  • “Can I quickly calculate today’s wind-chill / heat index before heading out?”
  • “Where do I report hail or damaging winds, and what details should I include?”

Color Schema

This document already uses the selected colors. Usage guidelines and CSS variables are defined below.

Background — --hsg-bg = #0b1220
Panels — --hsg-panel = #0f172a
Primary / CTA — --hsg-primary = #f59e0b
Accent / Links — --hsg-accent = #38bdf8
Text — --hsg-text = #e5e7eb

Usage:

  • Headings & key accents: --hsg-primary
  • Links & interactive focus: --hsg-accent
  • Body text: --hsg-text
  • Panels/cards: --hsg-panel on --hsg-bg

Typography

  • Headings: Oswald (500–600) – condensed, high-impact for titles and section headings.
  • Body: Inter (400–700) – modern, readable sans-serif for paragraphs and UI text.
  • Fallbacks: system UI stack for performance & resilience.

CSS / Design Notes

  • Color variables in :root ensure consistency across pages; dark theme supports nighttime readability.
  • Headings (Oswald) provide strong visual hierarchy; body text (Inter) optimizes readability.
  • Components use rounded corners and subtle borders for separation without reducing contrast.
  • Responsive grid: single column on mobile, two columns or more on larger screens.
